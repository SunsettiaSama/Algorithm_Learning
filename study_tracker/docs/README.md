# 学习进度追踪系统

一个强大的学习文件管理系统，帮助你追踪学习进度、管理复习计划。

## 简介

学习进度追踪系统（Study Tracker）是一个专为算法学习者设计的工具，能够：

- **自动追踪** 你的所有学习文件（Python、Markdown、文本等）
- **智能提醒** 识别需要复习的内容，避免遗忘
- **记录日志** 保存每日学习内容，便于回顾
- **可视化进度** 通过彩色标记和统计信息直观展示学习状态

## 核心功能

### 📊 文件追踪
- 自动扫描和监控所有学习文件
- 记录每个文件的创建时间、修改时间、更新历史
- 计算距离上次修改的天数

### 🎨 智能颜色标记
根据文件最后修改的天数，自动标记文件状态：
- 🟢 **绿色**：最近1天内更新或30天以上（已掌握）
- 🟡 **黄色**：1-3天未更新（需要逐渐复习）
- 🔴 **红色**：3-30天未更新（需要紧急复习）

### 📝 学习日志
- 每日记录学习内容
- 按日期组织和查询
- 便于学习总结和反思

### ⚠️ 紧急复习提醒
- 自动识别3-30天未复习的文件
- 启动时提示需要复习的文件数量
- 快速定位需要学习的内容

### 🎯 学习状态管理
- 标记文件为"已学习"、"已掌握"等状态
- 为每个文件添加学习笔记
- 导出学习进度报告

## 快速开始

### 启动应用

```bash
cd study_tracker
python main.py
```

### 第一次使用

1. **扫描目录** - 点击"扫描目录"按钮，系统将扫描所有学习文件
2. **查看进度** - 在文件列表中查看各文件的颜色标记和复习状态
3. **复习文件** - 点击红色文件查看详情，复习后点击"更新"刷新时间戳
4. **记录日志** - 点击"学习日志"按钮，记录今天学习的内容

## 数据存储

系统会在追踪目录创建 `.study_tracker/` 文件夹，包含：

- **database.json** - 所有文件的信息、状态和日志（JSON格式）
- **.studyignore** - 忽略规则配置（类似.gitignore）
- **report.txt** - 导出的学习进度报告

## 常见问题

**Q: 如何重置所有数据？**
A: 删除根目录下的 `.study_tracker` 文件夹，然后重新扫描

**Q: 支持哪些文件类型？**
A: 支持所有文件类型，系统会追踪所有文件的修改时间

**Q: 能追踪多个项目吗？**
A: 可以，为每个项目分别运行，各自维护独立的数据库

**Q: 如何忽略某些文件？**
A: 编辑 `.study_tracker/.studyignore` 文件，或在GUI中右键点击目录选择"忽略"
