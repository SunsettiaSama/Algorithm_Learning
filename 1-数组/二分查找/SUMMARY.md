### 二分查找核心：明确「查找目标（target）」是逻辑设计的根本
二分查找的核心差异本质源于「查找目标（target）的定义」：LeetCode 704（二分查找）的 target 是题目直接给定的明确值，查找逻辑围绕“找到数组中等于该 target 的元素下标”展开，核心是匹配“相等”条件；而 LeetCode 69（x 的平方根）的 target 是隐含的、需要满足特定约束的未知值（即最大的满足 k² ≤ x 的整数 k），查找逻辑并非匹配“相等”，而是寻找符合条件的边界值。

这种 target 定义的差异直接决定了后续逻辑：704 只需验证元素是否等于已知 target，找到即返回；69 则需要通过收缩区间锁定“满足约束的最大边界”，即便循环结束也需基于区间边界（而非直接匹配值）返回结果。而无论是哪种 target 定义，都需先明确「查找区间规则」（左闭右闭/左闭右开），再匹配对应的循环条件（left ≤ right / left < right）和区间收缩方式，这是避免边界错误的关键。


### 搜索插入位置返回left的完整逻辑总结
循环结束后返回`left`并非偶然，而是「插入位置定义」「区间收缩规则」「循环结束状态」三者层层推导的必然结果，核心逻辑可拆解为以下四层：

#### 1. 插入位置的本质定义（目标锚点）
插入位置的核心是：在有序数组中找到**第一个大于 target 的元素下标**（若所有元素都小于 target，则为数组长度）。
- 例：nums=[1,3,5,6]，target=2 → 第一个大于2的元素是3（下标1），插入位置=1；
- 例：nums=[1,3,5,6]，target=7 → 无大于7的元素，插入位置=数组长度4；
- 例：nums=[1,3,5,6]，target=0 → 第一个大于0的元素是1（下标0），插入位置=0。
二分查找的最终目标，就是让某个指针精准指向这个「第一个大于target的位置」。

#### 2. 左闭右闭区间的收缩逻辑（核心规则）
采用「左闭右闭 [left, right]」区间时，循环内的指针移动严格遵循以下规则，且会形成不可逆的核心结论：
| 条件                | 指针操作          | 操作本质                          |
|---------------------|-------------------|-----------------------------------|
| nums[mid] < target  | left = mid + 1    | 排除mid及左侧所有数（这些数都<target） |
| nums[mid] > target  | right = mid - 1   | 排除mid及右侧所有数（这些数都>target） |
| nums[mid] == target | return mid        | 找到目标，直接返回（无需后续判断）|

**核心结论1**：**整个循环过程中，始终保持「left 左侧的所有数都 < target」「right 右侧的所有数都 > target」。**
**这个结论是指针移动规则的直接结果——我们每一步都在把「小于target的数」划到left左边，把「大于target的数」划到right右边，从未打破这个边界。**

#### 3. 循环结束时的指针状态（关键推导）
循环条件为`left <= right`，因此循环终止的唯一条件是`left > right`；又因为每次指针移动仅±1（不会跨步），**所以终止时必然满足`left = right + 1`**。

结合「核心结论1」和终止状态，可推导出：
- **`right` 是「最后一个小于 target 的数的下标」（若所有数都大于target，right会被收缩到-1，即无小于target的数）；**
- **`left` 是「第一个大于 target 的数的下标」（若所有数都小于target，left会被收缩到数组长度，即无大于target的数）。**

**核心结论2**：循环结束时，`left` 恰好指向「第一个大于target的元素下标」，完全匹配插入位置的本质定义。

#### 4. 返回left的必然性（最终结论）
插入位置的定义是「第一个大于target的元素下标」，而循环结束时`left`正好指向这个位置，`right`则指向「最后一个小于target的下标」（`right = left - 1`）。
若返回`right`，得到的是「最后一个小于target的下标」（如target=2时right=0），与插入位置的要求完全不符；只有返回`left`，才能精准匹配插入位置的核心定义。

### 总结
1. 插入位置的本质是「第一个大于target的下标」，这是二分查找需要锁定的最终目标；
2. 左闭右闭区间的收缩规则保证了「left左侧全<target、right右侧全>target」，为定位目标奠定基础；
3. 循环结束时`left>right且left=right+1`，此时left恰好是「第一个大于target的下标」，因此返回left是唯一正确选择。
