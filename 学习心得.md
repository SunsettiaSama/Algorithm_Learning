## 一、算法题知识区间判断方法论（可落地执行）
### 1. 先看问题目标与数据规模
- 目标类型判断
  - 求「所有解/存在性」→ 优先考虑：枚举、回溯、搜索
  - 求「最优解/最值」→ 优先考虑：贪心、动态规划、分治
  - 求「最短路径/连通性」→ 优先考虑：图论、广搜/深搜
- 数据规模阈值（核心判断依据）
  - n ≤ 20 → 指数级算法可接受（枚举、回溯、全排列）
  - 20 < n ≤ 1000 → O(n²) 可接受（动态规划、部分枚举优化）
  - 1000 < n ≤ 1e5 → 线性/对数级算法（贪心、二分、哈希优化）
  - n > 1e5 → 必须用 O(n) 或 O(n log n) 算法（避免枚举）

### 2. 再看问题结构特征
- 可分解为重叠子问题 → 动态规划
- 每一步局部最优能推导出全局最优 → 贪心
- 需要遍历所有组合/排列 → 枚举/回溯
- 存在明确的邻接关系或状态转移 → 图论/搜索

### 3. 最后匹配典型题型特征
- 排列组合/子集生成 → 枚举/回溯
- 最值问题（如最大子数组、最小路径和）→ 动态规划/贪心
- 字符串匹配/暴力验证 → 枚举
- 图的连通性/最短路径 → 图论
