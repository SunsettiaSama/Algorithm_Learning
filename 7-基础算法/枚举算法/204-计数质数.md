# 埃拉托斯特尼筛法（埃式筛）- 计数质数最优解总结
## 一、埃式筛核心概念
埃拉托斯特尼筛法（Sieve of Eratosthenes）是一种**高效的批量质数筛选算法**，核心思想是“**先假设所有数为质数，再通过剔除合数的方式反向筛选出质数**”，是解决“统计n以内质数个数”类问题的最优解之一，广泛应用于数论相关的算法场景。

## 二、埃式筛核心思想（四步拆解）
1. **初始化标记数组**：创建一个长度为n的数组，用`1`（或`True`）标记“是质数”，`0`（或`False`）标记“非质数”，初始时默认所有数都是质数；
2. **边界预处理**：直接将0和1标记为非质数（因为0和1本身不符合质数定义）；
3. **核心筛选过程**：从最小的质数2开始，遍历到`√n`（关键优化），对于每个确认是质数的数`i`，将其所有倍数（从`i²`开始，避免重复剔除）标记为非质数；
4. **统计结果**：最终数组中剩余的`1`的数量，就是n以内的质数个数。

## 三、埃式筛的核心优势（对比纯枚举）
| 维度         | 纯枚举法       | 埃式筛         |
|--------------|----------------|----------------|
| 时间复杂度   | O(n√n)         | O(n log log n) |
| 核心逻辑     | 逐个判断是否为质数 | 批量剔除合数   |
| 效率表现     | 小范围（n<10⁴）可用，大范围卡顿 | 支持百万级/千万级数据快速筛选 |

## 四、关键优化点解释
1. **为什么遍历到√n？**
   若一个数`n`是合数，则它必然可以分解为两个因数`a×b`，且其中至少有一个因数≤√n。因此只需处理到√n，就能完成所有合数的剔除，无需遍历到n-1。
2. **为什么从i²开始标记倍数？**
   `i`的倍数中，小于`i²`的部分（如i=3时，6=2×3）已经被更小的质数（如2）标记过了，从`i²`开始标记可避免重复操作，进一步提升效率。

## 拆分后清晰易懂的埃式筛代码
原代码中`isPrime[i * i:n:i] = [0] * ((n - 1 - i * i) // i + 1)`这行逻辑较浓缩，以下将其拆分为**分步计算+详细注释**，保留核心逻辑的同时大幅提升可读性：

```python
def count_primes_eratosthenes(n: int) -> int:
    """
    埃拉托斯特尼筛法（埃式筛）：统计n以内的质数个数
    参数：
        n: 上限值（不包含n本身）
    返回：
        int: n以内的质数总个数
    """
    # 边界条件：小于2的数没有质数
    if n < 2:
        return 0
    
    # 步骤1：初始化标记数组，1表示是质数，0表示非质数
    # 数组长度为n，索引对应数字0~n-1
    is_prime = [1] * n
    
    # 步骤2：预处理0和1，它们不是质数
    is_prime[0] = 0  # 数字0非质数
    is_prime[1] = 0  # 数字1非质数
    
    # 步骤3：核心筛选逻辑，遍历到√n即可
    max_check = int(n ** 0.5) + 1  # +1是为了包含√n本身
    for current_num in range(2, max_check):
        # 仅处理当前是质数的数（非质数无需处理，其倍数已被剔除）
        if is_prime[current_num]:
            # 计算需要标记的起始位置：从current_num²开始（避免重复标记）
            start_pos = current_num * current_num
            # 计算需要标记的元素个数：确保覆盖到n-1，且数量准确
            # 公式推导：(结束位置 - 起始位置) // 步长 + 1
            mark_count = (n - 1 - start_pos) // current_num + 1
            # 将current_num的所有倍数（start_pos到n-1，步长current_num）标记为非质数
            is_prime[start_pos:n:current_num] = [0] * mark_count
    
    # 步骤4：统计剩余的质数数量（数组中1的个数）
    prime_count = sum(is_prime)
    return prime_count

# 测试用例（可直接运行验证）
if __name__ == "__main__":
    print(count_primes_eratosthenes(10))   # 输出4（质数：2,3,5,7）
    print(count_primes_eratosthenes(2))    # 输出0（无质数）
    print(count_primes_eratosthenes(100))  # 输出25（100以内有25个质数）
```

### 代码逻辑拆分说明
| 代码段                | 核心作用                                                                 |
|-----------------------|--------------------------------------------------------------------------|
| `is_prime = [1] * n`  | 初始化标记数组，用索引对应数字，值标记是否为质数                         |
| `is_prime[0/1] = 0`   | 预处理非质数边界，0和1直接标记为非质数                                   |
| `max_check = int(n**0.5)+1` | 计算筛选的最大边界，避免无效遍历                                         |
| `if is_prime[current_num]` | 仅处理质数的倍数，非质数的倍数已被更早的质数处理过                       |
| `start_pos = current_num * current_num` | 确定标记起始位置，避免重复剔除（如i=3时从9开始，而非6）|
| `mark_count = ...`    | 精准计算需要标记的元素个数，确保赋值时数组长度匹配，避免索引越界         |
| `is_prime[start_pos:n:current_num] = [0]*mark_count` | 批量标记当前质数的所有倍数为非质数                                       |
| `sum(is_prime)`       | 统计标记数组中剩余的1的数量，即最终的质数个数                             |

---

### 总结
1. 埃式筛的核心是**“假设-剔除”反向筛选**，通过批量标记合数替代逐个判断，效率远高于纯枚举；
2. 关键优化点是**遍历到√n** 和**从i²开始标记倍数**，既减少遍历次数，又避免重复操作；
3. 拆分后的代码将浓缩的赋值逻辑拆分为“起始位置计算+数量计算+批量标记”，更易理解每一步的目的，适合新手学习和调试。
