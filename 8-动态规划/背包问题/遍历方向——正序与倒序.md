



### 定义

$$dp[i][j] = \text{True 如果前 } i \text{ 个元素（下标 }0..i-1\text{）可以选出若干个数，其和恰好等于 } j，\text{否则 False}$$

其中 $ i = 0, 1, \dots, n$，$ j = 0, 1, \dots, target$。

**边界**：  

$$dp[0][0] = \text{True},\quad dp[0][j] = \text{False} \ (j>0)
$$

**状态转移**（考虑第 $i$ 个元素，其值为 $w = nums[i-1]$）：  

$$dp[i][j] = dp[i-1][j] \ \bigvee\ \big( j \ge w \ \text{且} \ dp[i-1][j-w] \big)
$$
其中 $\bigvee$ 表示逻辑或。

**空间优化——2D到1D**

观察发现，第 $i$ 行只依赖第 $i-1$ 行。我们可以只用一个一维数组 `dp[j]` 来代表“当前行的状态”。但在更新过程中，我们必须小心，不能丢失上一行的信息。

假设在处理第 $i$ 个元素之前，`dp[j]` 存储的是 **上一行** 的值，即 $dp[i-1][j]$。  
处理完第 $i$ 个元素后，我们希望 `dp[j]` 变成新行的值 $dp[i][j]$。

根据转移方程：

$$dp_{\text{new}}[j] = dp_{\text{old}}[j] \ \bigvee\ \big( j \ge w \ \text{且} \ dp_{\text{old}}[j-w] \big)$$

其中 $dp_{\text{old}}$ 是上一行（未处理当前元素）的值，$dp_{\text{new}}$ 是当前行（处理完当前元素）的值。


### 正序和倒序的决定其实很简单：
1. 如果我们的约束是，必须让背包丢弃重量为w的物品，加上当前价值，那么就是倒序，其可被表达为下式。


$$dp_{\text{new}}[j] = dp_{\text{old}}[j] \ \bigvee\ \big( j \ge w \ \text{且} \ dp_{\text{old}}[j-w] \big)
$$

- 可以注意到，因为更新的方向为j - w，那么此时如果采用正序，即从j-w到j本身，那么我们就试图用新的dp更新新的dp（而不是旧的dp更新新dp）

2. 反过来说，如果必须要求总重量大于w，那么更新的方向就完全反了过来，如下式

$$dp_{\text{new}}[j] = dp_{\text{old}}[j] \ \bigvee\ \big( j \ge w \ \text{且} \ dp_{\text{old}}[j+w] \big)$$

- 此时，必须让旧的dp包含j+w项，而新一轮的更新是从j开始，旧的在前面，可以拿来更新当前的dp
