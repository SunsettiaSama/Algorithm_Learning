# 平方和三元数（Count Square Sum Triples）解法总结
## 一、问题描述
### 题目核心
给定整数 `n`，统计 `1~n` 范围内满足 **毕达哥拉斯三元组** 条件 `a² + b² = c²` 的正整数三元组 `(a, b, c)` 的数量（`a、b、c` 均为 1 到 n 之间的正整数）。

### 示例
- 输入 `n = 5`，输出 `2`（有效三元组：`(3,4,5)`、`(4,3,5)`）；
- 输入 `n = 10`，输出 `4`（有效三元组：`(3,4,5)`、`(4,3,5)`、`(6,8,10)`、`(8,6,10)`）。

## 二、解法一：特殊优化解法（推荐）
### 标注
该解法为 **特殊解法**，核心是结合数学性质+枚举剪枝，将时间复杂度从暴力枚举的 `O(n³)` 优化至 `O(n²)`，是该问题的最优解之一。

### 核心思路
1. **数学性质简化**：利用“`c` 为斜边 → `c² ≤ n²`”“`(a,b,c)` 与 `(b,a,c)` 是对称解”的性质，减少遍历范围；
2. **两层循环替代三层**：放弃遍历 `c`，通过 `a² + b²` 计算 `c²`，再验证 `c` 是否为整数；
3. **提前剪枝**：当 `a² + b² > n²` 时，终止内层循环（后续 `b` 增大只会让和更大，无有效解）；
4. **对称补偿**：仅遍历 `b < a` 的组合，最后结果乘 2 补全 `b > a` 的对称解，避免重复遍历。

### 完整代码（Python）
```python
import math
from math import isqrt

class Solution:
    def countTriples(self, n: int) -> int:
        ans = 0
        # 遍历a：1 ~ n-1（a=n时a²=n²，b≥1会导致a²+b²>n²，无有效解）
        for a in range(1, n):
            # 遍历b：1 ~ a-1（固定b < a，减少一半遍历量）
            for b in range(1, a):
                a_square = a * a
                b_square = b * b
                sum_square = a_square + b_square
                
                # 提前剪枝：和超过n²，后续b更大，无需继续
                if sum_square > n * n:
                    break
                
                # 验证sum_square是否为完全平方数（c为整数）
                c = isqrt(sum_square)
                if c * c == sum_square:
                    ans += 1
        
        # 对称补偿：(a,b,c) 和 (b,a,c) 各算一次
        return ans * 2

# 测试用例
if __name__ == "__main__":
    sol = Solution()
    print(sol.countTriples(5))  # 输出2
    print(sol.countTriples(10)) # 输出4
```

### 关键细节说明
| 代码段                | 核心作用                                                                 |
|-----------------------|--------------------------------------------------------------------------|
| `range(1, n)`         | a最大取n-1，避免a=n时a²+b²必然超过n²                                     |
| `range(1, a)`         | 固定b < a，仅统计一组对称解，减少遍历量                                   |
| `sum_square > n*n`    | 提前剪枝，终止无效内层循环                                               |
| `isqrt(sum_square)`   | 整数平方根函数（无浮点数精度误差），验证sum_square是否为完全平方数        |
| `ans * 2`             | 补全(b,a,c)的对称解，无需重复遍历                                         |

## 三、解法二：一般暴力枚举解法
### 核心思路
最直观的枚举思路：三层循环遍历 `a、b、c`（均为 1~n 的正整数），逐一验证 `a² + b² = c²`，统计符合条件的三元组数量。

### 完整代码（Python）
```python
class Solution:
    def countTriples(self, n: int) -> int:
        count = 0
        # 三层循环遍历所有可能的a、b、c（1~n）
        for a in range(1, n + 1):
            for b in range(1, n + 1):
                for c in range(1, n + 1):
                    if a * a + b * b == c * c:
                        count += 1
        return count

# 测试用例
if __name__ == "__main__":
    sol = Solution()
    print(sol.countTriples(5))  # 输出2
    print(sol.countTriples(10)) # 输出4
```

### 优缺点分析
| 维度         | 优点                     | 缺点                                   |
|--------------|--------------------------|----------------------------------------|
| 理解成本     | 逻辑简单，新手易上手     | 无优化，效率极低                       |
| 时间复杂度   | -                        | O(n³)，n≥100时明显卡顿                 |
| 遍历效率     | -                        | 全量遍历，包含大量无效组合（如c<a/b）|
| 重复处理     | -                        | 重复遍历(a,b)和(b,a)，无剪枝           |

## 四、两种解法对比
| 对比维度       | 特殊优化解法（推荐）| 一般暴力枚举解法 |
|----------------|---------------------------|------------------|
| 时间复杂度     | O(n²)（两层循环+剪枝）| O(n³)（三层循环） |
| 遍历次数       | 仅遍历b < a的组合，剪枝减少无效计算 | 全量遍历所有a、b、c |
| 精度问题       | isqrt整数验证，无误差     | 无（但效率低）|
| 理解成本       | 中等（需理解数学性质）| 极低（纯暴力）|
| 适用场景       | 所有n（尤其n≥100时）| 仅n≤50的小范围场景 |

## 五、核心总结
1. **特殊解法的核心价值**：枚举算法的优化核心是「结合问题数学性质缩小解空间」——利用“斜边最大”“对称解”等性质减少遍历层数/次数，通过“提前剪枝”终止无效计算，是暴力枚举→高效解法的典型思路；
2. **一般解法的定位**：适合新手理解问题本质，但因时间复杂度高，仅适用于小范围数据；
3. **关键优化技巧**：
   - 减少循环层数（用数学计算替代一层循环）；
   - 提前剪枝（终止无效循环）；
   - 利用对称性减少遍历量（最后补偿）；
   - 优先使用整数运算（避免浮点数精度问题）。
